# Incident Reports

**Status:** Coming post-Network+ exam (June 2026)

---

## What This Section Is

Simulated helpdesk tickets written in standard IT support format. Each incident documents a realistic scenario from initial report to resolution — the same format used in real ticketing systems like ServiceNow, Jira Service Management, and osTicket.

---

## Ticket Format

Each incident report follows this structure:

```
Ticket #:     INC001
Date:         YYYY-MM-DD
Category:     Account Management / Hardware / Network / Software / Access
Priority:     Low / Medium / High / Critical
Reported By:  [Simulated user]

Description:
  What the user reported

Steps Taken:
  1. What was checked first
  2. What was ruled out
  3. What was attempted

Resolution:
  What fixed it

Root Cause:
  Why it happened

Time to Resolve:
  X minutes / hours

Prevention:
  What could prevent this in the future
```
